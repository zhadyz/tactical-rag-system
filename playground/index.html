Tips & Best Practices

Start with the pre-loaded examples and modify them to match your use case. This is faster than writing from scratch!

For Best Results

  1. Use Descriptive Questions: Clear, specific questions get better answers
  2. Enable GPU When Available: Significantly faster processing
  3. Adjust max_chunks: More chunks = better context, but slower
  4. Monitor Metrics: Use latency and token counts to optimize

Common Patterns

# Pattern 1: Quick Answer
response = requests.post(API_URL, json={
    "question": "Your question here",
    "mode": "simple"
})
 
# Pattern 2: High-Quality Answer
response = requests.post(API_URL, json={
    "question": "Your question here",
    "mode": "advanced",
    "use_gpu": True,
    "max_chunks": 15
})
 
# Pattern 3: Fast Lookup
response = requests.post(API_URL, json={
    "question": "Your question here",
    "mode": "simple",
    "max_chunks": 3
})

Error Handling

The playground provides helpful error messages:

  • Network Errors: Connection issues with the API
  • Syntax Errors: Code syntax problems
  • API Errors: Invalid parameters or rate limits
  • Timeout Errors: Requests taking too long

If you encounter rate limiting, wait a few moments before trying again. Consider implementing exponential backoff in production code.
